# 需求規格說明書

> ZF_TrendPicking 技術型態篩選系統

## 1. 系統概述

ZF_TrendPicking 是一套自動化股票技術型態篩選系統，支援**台股**與**美股**兩個完全獨立的市場。系統每日盤後自動抓取個股股價，透過技術指標演算法篩選符合條件的股票，並將結果匯出至 Google Sheet 供使用者查閱。

### 1.1 系統目標

- 自動篩選兩種技術型態：**VCP 強勢股**與**三線開花**
- 每日盤後自動執行，無需人工介入
- 結果輸出至 Google Sheet，方便即時查閱
- 台股與美股完全隔離，互不影響

## 2. 功能需求

### 2.1 主檔管理

| 項目 | 台股 | 美股 |
|------|------|------|
| 排程頻率 | 每月 1 日 | 每月 1 日 |
| 資料來源 | FinMind API | NASDAQ FTP |
| 範圍 | 上市櫃（不含興櫃、創新板） | 全部美股（排除 ETF） |
| 更新方式 | 每月全量更新（DB 用 INSERT OR REPLACE upsert） | 每月全量更新（DB 用 INSERT OR REPLACE upsert） |
| 排序 | Google Sheet 每次清空後依代號升冪重寫全部資料 | Google Sheet 每次清空後依代號升冪重寫全部資料 |

**主檔欄位**：代號、股名、公司名、產業分類1、產業分類2（optional）、產品組合

> 無資料的欄位填入 `-`

### 2.2 VCP 強勢股篩選

每日盤後撈取個股股價（台股 FinMind + yfinance 備援、美股 yfinance），排除 ETF 等非普通股商品，執行以下篩選邏輯：

#### 2.2.1 強勢清單（3 條件 AND）

1. **均線多頭排列**：個股今日**收盤價** > 50 日均價 > 150 日均價 > 200 日均價（均線皆用收盤價計算）
2. **MA200 趨勢向上**：近 200 天均價 大於 20 天前的 200 日均價
3. **打敗大盤**：個股近 20 日漲跌幅（收盤價）> 大盤近 20 日漲跌幅

> 三個條件全部滿足 → 標記為強勢清單

#### 2.2.2 新高清單（2 條件 AND）

1. **接近 52 週新高**：個股近 5 日**最高價** ≈ 近 260 日（52 週）**最高價**（正負差距 ≤ 1%）
2. **打敗大盤**：個股近 20 日漲跌幅（收盤價）> 大盤近 20 日漲跌幅

> 兩個條件全部滿足 → 標記為新高清單

#### 2.2.3 最終結果

- **聯集**合併強勢清單與新高清單
- 依「近 20 日股價漲幅」降冪排序
- 符合強勢清單者在對應欄位標記 `O`
- 符合新高清單者在對應欄位標記 `O`

#### 2.2.4 大盤報酬計算

- 台股：計算加權指數（TAIEX）近 20 日漲跌幅
- 美股：計算 S&P 500（^GSPC）近 20 日漲跌幅
- 此數值作為暫存值（temp value），供 VCP 篩選條件比對使用

### 2.3 三線開花篩選

每日盤後撈取個股股價（同 VCP 資料來源），執行以下篩選邏輯：

#### 2.3.1 篩選條件（2 條件 AND）

1. **短期均線多頭排列**：個股今日**收盤價** > 8 日均價 > 21 日均價 > 55 日均價（均線皆用收盤價計算）
2. **創 55 日收盤新高**：個股今日**收盤價** 為 55 日內的最高**收盤價**

#### 2.3.2 輸出計算

- **差距比例** = （今日股價 / 55 日內次高價）- 1
- 依「差距比例」降冪排序

## 3. Google Sheet 輸出規格

### 3.1 八個（台股 4 + 美股 4）Google Sheet 檔案

| # | 檔案名稱 | Sheet ID 環境變數 | 說明 |
|---|---------|-----------------|------|
| 1 | 台股公司主檔 | `SHEET_ID_COMPANY_MASTER` | 台股公司資訊 |
| 2 | 美股公司主檔 | `US_SHEET_ID_COMPANY_MASTER` | 美股公司資訊 |
| 3 | 台股VCP | `SHEET_ID_TW_VCP` | 台股 VCP 每日篩選結果 |
| 4 | 美股VCP | `US_SHEET_ID_VCP` | 美股 VCP 每日篩選結果 |
| 5 | 台股三線開花 | `SHEET_ID_TW_SANXIAN` | 台股三線開花每日篩選結果 |
| 6 | 美股三線開花 | `US_SHEET_ID_SANXIAN` | 美股三線開花每日篩選結果 |
| 7 | 台股驗證 | `SHEET_ID_VERIFICATION` | 台股每日完整計算過程驗證 |
| 8 | 美股驗證 | `US_SHEET_ID_VERIFICATION` | 美股每日完整計算過程驗證 |

### 3.2 分頁規則

#### 公司主檔（兩個獨立試算表）

**台股公司主檔試算表**（`SHEET_ID_COMPANY_MASTER`）

| 分頁順序 | 分頁名稱 | 內容 |
|---------|---------|------|
| 1 | 台股公司主檔 | 台股上市櫃公司資訊（固定分頁，每月清空後依代號升冪全量重寫） |
| 2 | 台股更新紀錄 | A1 欄位寫入更新時間 |

**美股公司主檔試算表**（`US_SHEET_ID_COMPANY_MASTER`）

| 分頁順序 | 分頁名稱 | 內容 |
|---------|---------|------|
| 1 | 美股公司主檔 | 美股公司資訊（固定分頁，每月清空後依代號升冪全量重寫） |
| 2 | 美股更新紀錄 | A1 欄位寫入更新時間 |

#### VCP / 三線開花（台股+美股各自獨立）

- 每日新增分頁，分頁名稱格式：`YYMMDD`
- 新分頁固定插入在**第二個分頁位置**

### 3.3 欄位定義

#### VCP 輸出欄位

| 欄位 | 說明 |
|------|------|
| 代號 | 股票代號 |
| 股名 | 股票名稱 |
| 公司名 | 公司全名 |
| 產業分類1 | 主要產業分類 |
| 產業分類2 | 次要產業分類（optional） |
| 產品組合 | 公司主要產品 |
| 近20日股價漲幅 | 近 20 個交易日的漲跌幅 |
| 強勢清單 | 符合則標記 `O` |
| 新高清單 | 符合則標記 `O` |

#### 三線開花輸出欄位

| 欄位 | 說明 |
|------|------|
| 代號 | 股票代號 |
| 股名 | 股票名稱 |
| 公司名 | 公司全名 |
| 產業分類1 | 主要產業分類 |
| 產業分類2 | 次要產業分類（optional） |
| 產品組合 | 公司主要產品 |
| 今日股價 | 今日收盤價 |
| 55日內次高價 | 55 個交易日內第二高收盤價 |
| 差距比例 | (今日股價 / 55日次高價) - 1 |

### 3.4 驗證資料匯出

系統另提供驗證用的 Google Sheet（`SHEET_ID_VERIFICATION` / `US_SHEET_ID_VERIFICATION`），匯出當日全部股票的完整計算過程（收盤價、各週期均線、高低點、報酬率、各篩選條件布林值等），供人工核對篩選邏輯正確性。

每日產生兩個頁籤：`YYMMDD_VCP`（20 欄）與 `YYMMDD_三線`（14 欄）。

## 4. 錯誤處理需求

### 4.1 HTTP 錯誤分級

| 錯誤類型 | 狀態碼 | 處理方式 |
|---------|--------|---------|
| 不可重試錯誤 | 400, 401, 403, 404（4XX 除 429） | 立即中斷，拋出異常，記錄 Error Log |
| 可重試錯誤 | 任一 5XX（狀態碼 ≥ 500）或 429 | 進入重試排程 |

### 4.2 重試機制

| 次數 | 等待時間 |
|------|---------|
| 第 1 次 | 5 分鐘（300 秒） |
| 第 2 次 | 10 分鐘（600 秒） |
| 第 3 次 | 1 小時（3600 秒） |

- 最大重試次數：3 次（可從設定檔調整，環境變數 `MAX_RETRIES`）
- 每次重試記錄 `time`、`status_code`、`retry_count`、`params`

### 4.3 Log 紀錄

- 重試紀錄寫入 Google Sheet「台股更新紀錄」/「美股更新紀錄」分頁
- 同時寫入本地日誌檔案：
  - 台股：`logs/zf_trend.log`
  - 美股：`logs/zf_trend_us.log`

## 5. 非功能需求

### 5.1 台股/美股隔離

- 兩個市場使用**完全獨立**的資料庫、設定、日誌、Google Sheet
- 一個系統故障不影響另一個
- 各自獨立的 GitHub Actions 排程

### 5.2 自動排程

| 任務 | 頻率 | 台灣時間 | UTC |
|------|------|---------|-----|
| 台股每日篩選 | 週一至週五 | 17:45 | 09:45 |
| 美股每日篩選 | 週一至週五 | 05:30 | 21:30 |
| 台股每月更新 | 每月 1 日 | 09:00 | 01:00 |
| 美股每月更新 | 每月 1 日 | 09:30 | 01:30 |

### 5.3 資料庫永久備份

- 主要備份：GitHub Release（永久保存，gzip 壓縮）
- 備援備份：GitHub Artifact（90 天有效，快速存取）
- 每次任務完成後自動備份

### 5.4 交易日判斷

- 自動跳過非交易日（假日、國定假日）
- 支援 `--force` 旗標強制執行
- 台股與美股各自維護獨立的交易日曆（2024-2026）
